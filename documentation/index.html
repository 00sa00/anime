anime.js

Targets

CSS Selector

Can be any CSS selector.
Note that pseudo elements cannot be animated in JavaScript.

Accepts Examples
string '.item'
array ['.item', '#thing']

DOM Node / NodeList

Can be any DOM Node or NodeList.

Accepts Examples
DOM Node el.querySelector('.item')
NodeList el.querySelectorAll('.item')
array [el.querySelector('.item'), el.getElementById('#thing')]

JavaScript Object

A javascript Object with at least one property containing a numerical value.

Accepts Examples
Object const obj = { prop: '0%' }
array [obj, obj2, obj3] 
{"charged":"0%","cycles":120}

Mixed Array

An array containing different type of targets

Accepts Examples
array ['.item', document.getElementById('#thing')]

Values

Unitless

If the original value has a unit, it will be automatically added to the animated value.

Specific unit

Force the animation to use a certain unit.
It's important to note that anime will not automatically convert units.
For example, if the target initial value is '100px' and your animation value is '3rem', the animation will starts at '100rem'.
If you need your animation to starts from a specific value, check out the From To values section.

Relative

Add, substract or multiply the original value.
Default units are preserved.

Accepts Effect
'+=' Add
'-=' Substract
'*=' Multiply

Timeline

Basic timeline

Timelines let you synchronise multiple animations together.
By default each child animation will start after the previous one ends. 

var myTimeline = anime.timeline(parameters);
Argument Type Info Required
parameters Object The default parameters of the timeline inherited by children Optional 
myTimeline.add(parameters, offset);
Argument Types Info Required
parameters Object The child animation parameters, override the timeline default parameters Required
time offset String or Number Check out the Timeline offsets section Optional 
var myTimeline = anime.timeline(parameters);
myTimeline.add(parameters);

Time Offsets

Defines when the child aniamtion starts.

Offset Example Info
'+=' '+=200' Starts 200ms after the previous animation ends
'-=' '-=200' Starts 200ms before the previous animation ends
Number 100 Starts at 100ms, reagardless of the animtion position in the timeline
no offset
relative offset ('-=600')
absolute offset (400)

Parameters inheritance

Write less code by defining default parameters.
Every child inherits the default parameters of the timeline.
Default parameters can be overrided.

Controls

Play / Pause

Manualy play or pause the animation.

var animation = anime({
  targets: '.item',
  translate: 250,
  autoplay: false
});

animation.play();
animation.pause();

Restart

Restart the animation from the default values

var animation = anime({
  targets: '.item',
  translate: 250
});

animation.restart();

Reverse

Reverse the animation direction

var animation = anime({
  targets: '.item',
  translate: 250
});

animation.reverse();

Seek

Jump to a specific time of the animation

var animation = anime({
  targets: '.item',
  translate: 250
});

animation.seek(500);

Timeline controls

Timelines can be controled like any other anime instance

States

Adding states

States let you define and store multiple set of properties for one animation.

Go directly to a specific state : 

animation.states.goTo(stateName);

Or animate to a state : 

animation.states.animateTo(stateName);

Parameters inheritance

By default, only the properties specified in the state will be changed.
You can specify default properties that all states will inherit.
You can override these properties when defining or animating a state.

Callbacks & Promises

Update

update() callback is triggered on every frames as soon as the animation starts playing.

Accepts Parameters Info
Function animation Return the current animation Object

Begin

begin() callback is triggered when the animation.currentTime is equal or superior to the animation.delay.
It's also called if the animation duration is 0.

Accepts Parameters Info
Function animation Return the current animation Object

Run

run() callback is triggered on every frames if the animation.currentTime is equal or superior to the animation.delay.

Accepts Parameters Info
Function animation Return the current animation Object

Complete

complete() callback is triggered when the animation finished.

Accepts Parameters Info
Function animation Return the current animation Object

Finished promise

Every animation instances return a finished promise when the animation finised.

Note that Promises are not suported in IE < 11.
